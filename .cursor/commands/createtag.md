<!-- ## 🧩 提示词模板：生成发布 Tag 信息与命令（含类型分类 + 版本自动递增） -->

**任务目标：**  
分析最后一个 tag 与 `master` 分支最新提交之间的改动，生成一个新的发布 tag 信息和对应的 `git` 命令。


---

### 版本递增规则
- 版本号格式：`v<major>.<minor>.<patch>`
- 默认规则：`patch + 1`

---

### 生成要求
- 自动按规则计算下一个版本号  
- 标题格式：`Release vX.Y.Z`  
- 内容结构必须包含：
  - **分类汇总**（明确区分更新类型）：
    - **feat**：新功能或增强  
    - **fix**：问题修复或错误修正  
    - **refactor / perf / chore**：结构优化、性能提升、构建/依赖更新  
  - 每类下 1～3 条简短说明（总计 4～6 条）  
  - **变更统计**：文件数、增减行数、提交数  
- 输出格式清晰、风格统一，末尾附 `git push` 指令  

---

### 示例参考

#### 示例 1
```sh
git tag -a v1.13.9-stable -m "Release v1.13.9

feat:
- 登录态恢复支持跨标签页同步
- 新增异常工单识别机制

fix:
- 报表导出格式与时间精度问题
- 客户看板筛选逻辑错误修复

perf/refactor:
- 缓存刷新策略优化
- 接口响应时间降低 15%

变更统计：24文件 +1,046/-382行 11提交"

git push origin v1.13.9
```

请注意: 不要帮我执行
