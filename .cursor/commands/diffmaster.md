
**任务目标：**
分析一下当前分析相对于 master 的提交信息和文件修改统计，并生成一段Merge Request的信息


---

### 生成要求
- 标题格式：`Merge Request 分支名`
- 内容结构必须包含：
  - **分类汇总**（明确区分更新类型）：
    - **feat**：新功能或增强
    - **fix**：问题修复或错误修正
    - **refactor / perf / chore**：结构优化、性能提升、构建/依赖更新
  - 每类下 1～3 条简短说明（总计 4～6 条）
  - **变更统计**：文件数、增减行数、提交数
- 输出格式清晰、风格统一

---

### 示例参考

#### 示例 1
```sh
Merge Request

feat:
- 登录态恢复支持跨标签页同步
- 新增异常工单识别机制

fix:
- 报表导出格式与时间精度问题
- 客户看板筛选逻辑错误修复

perf/refactor:
- 缓存刷新策略优化
- 接口响应时间降低 15%

变更统计：24文件 +1,046/-382行 11提交"
```
